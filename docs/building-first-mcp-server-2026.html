1. What is MCP and Why Build a Server?

The Model Context Protocol (MCP) is an open protocol developed by Anthropic that enables AI models to connect securely with external data sources and tools. Unlike traditional tool-calling where every integration is custom-built, MCP provides a standardized interface that works across any AI model and any tool provider.

If you've ever integrated APIs into AI applications, you know the pain: every new tool requires custom code, error handling, and maintenance. MCP solves this by creating a universal "USB port" for AI — one connection, infinite tools.

Building your own MCP server lets you:

  • Expose your APIs, databases, or services to any MCP-compatible AI client
  • Monetize your data/tools through the growing MCP ecosystem
  • Build once, use everywhere — no more per-client integration work
  • Join 23,000+ MCP servers already in production (Glama.ai registry)

3. Method 1: Python MCP Server

Step 1: Install the MCP SDK

# Create a virtual environment
python3 -m venv mcp-env
source mcp-env/bin/activate  # On Windows: mcp-env\Scripts\activate

# Install the MCP Python SDK
pip install mcp httpx

Step 2: Create Your First Server

Create a file called server.py:

import mcp.server.stdio
import mcp.types as types
from mcp.server import Server
import asyncio

# Create a server instance
server = Server("my-first-mcp-server")

@server.list_tools()
async def list_tools() -> list[types.Tool]:
    """Define the tools this server provides."""
    return [
        types.Tool(
            name="get_weather",
            description="Get current weather for a city",
            inputSchema={
                "type": "object",
                "properties": {
                    "city": {"type": "string", "description": "City name"}
                },
                "required": ["city"]
            }
        )
    ]

@server.call_tool()
async def call_tool(name: str, arguments: dict) -> list[types.TextContent]:
    """Handle tool calls from AI clients."""
    if name == "get_weather":
        city = arguments["city"]
        # Replace with actual weather API call
        return [types.TextContent(
            type="text",
            text=f"🌤️ The weather in {city} is 22°C with clear skies."
        )]
    raise ValueError(f"Unknown tool: {name}")

async def main():
    async with mcp.server.stdio.stdio_server() as (read_stream, write_stream):
        await server.run(
            read_stream,
            write_stream,
            server.create_initialization_options()
        )

if __name__ == "__main__":
    asyncio.run(main())

Step 3: Test It

python server.py
# Server starts and listens on stdio
# Test with: echo '{"jsonrpc":"2.0","method":"initialize","params":{},"id":1}' | python server.py

4. Method 2: TypeScript MCP Server

Step 1: Initialize Project

mkdir my-mcp-server && cd my-mcp-server
npm init -y
npm install @modelcontextprotocol/sdk zod

Step 2: Create the Server

Create src/index.ts:

import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import {
  CallToolRequestSchema,
  ListToolsRequestSchema,
} from "@modelcontextprotocol/sdk/types.js";

const server = new Server(
  { name: "my-mcp-server", version: "1.0.0" },
  { capabilities: { tools: {} } }
);

server.setRequestHandler(ListToolsRequestSchema, async () => {
  return {
    tools: [
      {
        name: "search_database",
        description: "Search a database for records matching a query",
        inputSchema: {
          type: "object",
          properties: {
            query: { type: "string", description: "Search query" },
            limit: { type: "number", description: "Max results (default: 10)", default: 10 }
          },
          required: ["query"]
        }
      }
    ]
  };
});

server.setRequestHandler(CallToolRequestSchema, async ({ params }) => {
  const { name, arguments: args } = params;
  
  if (name === "search_database") {
    const { query, limit = 10 } = args as { query: string; limit?: number };
    // Implement your database search logic here
    return [{
      type: "text",
      content: `Found ${limit} results for "${query}": [simulated results]`
    }];
  }
  
  throw new Error(`Unknown tool: ${name}`);
});

async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error("MCP server running on stdio");
}

main().catch(console.error);

5. Defining Tools and Resources

MCP supports three primary capability types:

CapabilityUse CaseExample
ToolsActions the AI can performSend email, query DB, call API
ResourcesData the AI can readFile contents, API responses, configs
PromptsReusable prompt templatesCode review template, email draft

Adding Resources

# Python: Adding resources
@server.list_resources()
async def list_resources() -> list[types.Resource]:
    return [
        types.Resource(
            uri="file:///config/app.json",
            name="app_config",
            description="Application configuration",
            mimeType="application/json"
        )
    ]

@server.read_resource()
async def read_resource(uri: str) -> str:
    if uri == "file:///config/app.json":
        return json.dumps({"debug": True, "version": "1.0.0"})
    raise ValueError(f"Unknown resource: {uri}")

6. Adding Authentication

⚠️ Security Note: Never expose sensitive APIs without authentication. MCP supports API Key, OAuth 2.0, and mTLS. See our MCP Authentication & Security Guide for production security patterns. 
# Python: API Key authentication example
class AuthenticatedServer:
    def __init__(self, api_key: str):
        self.api_key = api_key
    
    def validate_request(self, headers: dict) -> bool:
        provided_key = headers.get("Authorization", "").replace("Bearer ", "")
        return provided_key == self.api_key

# Usage: Only process requests with valid auth
@server.call_tool()
async def call_tool(name: str, arguments: dict, headers: dict) -> list[types.TextContent]:
    auth = AuthenticatedServer(os.environ["API_KEY"])
    if not auth.validate_request(headers):
        raise PermissionError("Invalid API key")
    # ... proceed with tool execution

Integration Testing

# Python: pytest integration test
import pytest
from server import server

@pytest.mark.asyncio
async def test_list_tools():
    """Test that tools are properly listed."""
    tools = await list_tools()
    assert len(tools) > 0
    assert any(t.name == "get_weather" for t in tools)

@pytest.mark.asyncio
async def test_call_tool():
    """Test tool execution."""
    result = await call_tool("get_weather", {"city": "Tokyo"})
    assert "Tokyo" in result[0].text
    assert "22°C" in result[0].text

8. Deployment Options

Option A: Local Development (stdio)

The simplest method — server communicates via stdin/stdout. Perfect for local testing and development. Configure your AI client (Claude Desktop, Cursor, etc.) to point to your local script.

Option B: HTTP/SSE Transport (Production)

from mcp.server.sse import SSEServer

sse_server = SSEServer(
    server,

9. Best Practices

  • Validate all inputs — Use Zod (TS) or Pydantic (Python) to validate tool arguments before processing
  • Handle errors gracefully — Return structured error messages, not raw exceptions
  • Rate limit your tools — Protect against abuse with token bucket or sliding window rate limiting
  • Log for debugging — Track tool calls, durations, and errors for observability
  • Version your protocol — Increment version when changing tool schemas to avoid client breakage
  • Use environment variables — Never hardcode API keys or secrets
  • Write comprehensive schemas — Detailed descriptions and examples in your tool schemas dramatically improve AI understanding
