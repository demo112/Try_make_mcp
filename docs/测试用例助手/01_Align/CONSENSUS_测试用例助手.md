# 需求共识文档：TestCase Assistant

## 1. 需求描述
构建一个离线可用的 MCP 服务，辅助测试人员快速编写测试用例。它不依赖大模型（因为是 Server 端），而是提供**规则化**的工具，让 Client 端的大模型（如果局域网内有部署本地大模型）或者直接由用户通过 MCP Client 调用来加速工作。

**特别说明**：MCP Server 本身不包含 LLM。它提供的工具是给 LLM 调用的。如果用户的局域网内没有 LLM，那么这个 MCP Server 将作为“规则库”被用户直接调用（例如通过简单的 CLI 或 GUI 包装）。
*考虑到用户提到“认可大模型”，假设局域网内可能部署了本地大模型（如 Llama3, Qwen 等）或者用户希望通过这个工具展示 MCP 的能力。*

## 2. 验收标准
1.  **独立运行**：提供 `.exe` 文件，在无 Python 环境的 Windows 上运行。
2.  **模板生成**：输入“登录功能”，返回包含正常流和异常流的标准 CSV/Markdown 格式用例。
3.  **边界值推荐**：输入“1-100的整数”，返回 `0, 1, 50, 100, 101, -1, null` 等测试值。
4.  **Mock 数据**：(可选) 生成一些测试用的 Mock 数据。

## 3. 技术实现
- **语言**: Python
- **库**: `mcp`, `faker` (用于生成测试数据)
- **打包**: `pyinstaller`

## 4. 任务边界
- 不包含用例管理系统 (TMS) 的数据库交互。
- 不包含实际执行测试的功能。
