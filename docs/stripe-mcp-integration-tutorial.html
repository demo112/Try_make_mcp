Stripe + MCP Integration Tutorial: Add Payments to Your MCP Server

Production-Verified Stripe Checkout Freemium Model

8 min read · Updated May 2026 · Based on BGPT MCP's real $0.02/result model

This tutorial shows you how to add Stripe payments to your MCP server using the freemium model — the exact pattern used by BGPT MCP (19 stars, active Stripe integration). Users start free, then upgrade when they need more.

Overview: The Freemium MCP Pattern

Key Insight: You don't need any changes to the MCP protocol spec to add payments. The API key is passed as a tool call parameter, completely transparent to the MCP client. — tadasant (MCP maintainer)

The pattern works like this:

  1. Free tier: No API key required. Users connect and use tools immediately. Rate-limited by IP (e.g., 10 calls/day).
  2. Free tier exhausted: Server returns JSON-RPC error code -32042 ("Payment required") with a subscribe URL.
  3. Upgrade: User visits your Stripe Checkout page, pays, receives an API key.
  4. Paid tier: User adds api_key parameter to their tool calls. Unlimited access.

Step 2: Add Freemium Gate to Your MCP Server

Here's the complete freemium gate logic. This follows the BGPT MCP pattern — API key is a tool call parameter, not an auth header.

2.1 API Key Validation

// validateApiKey(apiKey, env)
// Returns: { valid: boolean, tier: 'free' | 'pro' }
function validateApiKey(apiKey, env) {
  if (!apiKey) return { valid: false, tier: 'free' };

  // Check against comma-separated list in environment variables
  const validKeys = (env?.VALID_API_KEYS || '').split(',')
    .map(k => k.trim()).filter(k => k.length > 0);

  if (validKeys.includes(apiKey)) {
    return { valid: true, tier: 'pro' };
  }

  return { valid: false, tier: 'free' };
}

2.2 Rate Limiting (Free Tier)

// Using Cloudflare KV for persistent rate limiting
// Free tier: 10 calls per day per IP
async function checkFreeTierLimit(ip, env) {
  const today = new Date().toISOString().split('T')[0];
  const key = `usage:${ip}:${today}`;

  const count = parseInt(await env.USAGE_KV.get(key) || '0');

  if (count >= 10) return false; // Limit reached

  await env.USAGE_KV.put(key, String(count + 1), {
    expirationTtl: 86400 // Auto-cleanup after 24h
  });

  return true; // Still within limit
}

2.3 The Payment Required Error

// In your tools/call handler:
case 'tools/call': {
  const { name: toolName, arguments: args } = params;
  const apiKey = args.api_key; // BGPT pattern!
  const clientIP = request.headers.get('CF-Connecting-IP');

  const auth = validateApiKey(apiKey, env);

  if (auth.tier === 'pro') {
    // ✅ Paid user — execute directly
    result = await executeTool(toolName, args);
  } else {
    // Free tier — check rate limit
    const allowed = await checkFreeTierLimit(clientIP, env);
    if (allowed) {
      result = await executeTool(toolName, args);
    } else {
      // ❌ Payment required — MCP error code -32042
      isError = true;
      result = {
        code: -32042,
        message: 'Payment required: Daily free limit reached',
        data: {
          limit: 10,
          subscribe_url: '/subscribe',
          tip: 'Add api_key parameter for unlimited access'
        }
      };
    }
  }
  break;
}
Why -32042? This is the error code proposed in MCP Discussion #2436 for "Payment Required." It's not yet in the spec, but it's becoming the de facto standard.

Step 3: Handle Stripe Webhooks (Provision API Keys)

When a user completes checkout via your Payment Link, Stripe sends a webhook. Handle it to auto-provision API keys.

3.1 Add a Webhook Endpoint to Your Worker

// In your Cloudflare Worker fetch handler:
if (url.pathname === '/webhook' && request.method === 'POST') {
  const body = await request.text();
  const sig = request.headers.get('stripe-signature');

  // Verify webhook signature (optional but recommended)
  // const event = stripe.webhooks.constructEvent(body, sig, endpointSecret);

  const event = JSON.parse(body);

  if (event.type === 'checkout.session.completed') {
    const session = event.data.object;
    const customerEmail = session.customer_details?.email;
    const amount = session.amount_total;

    // Generate API key
    const apiKey = 'sk_live_' + crypto.randomUUID().replace(/-/g, '');

    // Store: apiKey → { email, credits, created }
    await env.USERS_KV.put(apiKey, JSON.stringify({
      email: customerEmail,
      credits: amount, // 1 credit = 1 cent
      created: new Date().toISOString(),
    }));

    // Send API key to user (email, or redirect with key in URL)
    console.log(`Provisioned ${apiKey} for ${customerEmail}`);
  }

  return new Response(JSON.stringify({ received: true }), {
    headers: { 'Content-Type': 'application/json' }
  });
}
Security: Always verify the stripe-signature header in production. Use the Stripe secret key to construct the event. This prevents attackers from sending fake webhook events.

Alternative: x402 HTTP Payment Protocol

Beyond Stripe, the MCP community is exploring x402 — an HTTP-native payment protocol using USDC on Base blockchain:

// x402 flow (no Stripe account needed!)
// 1. Client sends request
// 2. Server responds with 402 Payment Required + payment details
// 3. Client's wallet pays automatically
// 4. Server receives payment and processes the request

APIbase.pro uses x402 + MPP (Metered Payment Protocol) with 576 tools and 177 providers. Pricing ranges from $0.001–$1.00/call in USDC.

Recommendation: Start with Stripe (largest audience, easiest setup). Add x402 later if you want crypto-savvy users.

Security Checklist

  • ✅ Verify Stripe webhook signatures — prevent fake payment events
  • ✅ Rate limit free tier by IP — prevent abuse
  • ✅ Use CF-Connecting-IP — Cloudflare provides real client IPs
  • ✅ Don't expose Stripe secret keys — use wrangler secret put
  • ✅ Set TTL on KV usage records — auto-cleanup to control costs
  • ⚠️ Prompt injection risk — AI agents might forward payment params. Validate server-side.
  • ⚠️ API key in tool params — keys are visible to the AI model. Consider this in your threat model.
