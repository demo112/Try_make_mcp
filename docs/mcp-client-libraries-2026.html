Official MCP SDKs from Anthropic

Anthropic maintains official MCP SDKs for the most popular languages. These are the most thoroughly tested and recommended for production use.

Official Python SDK: mcp

pip install mcp
import mcp

# Create an MCP client
client = mcp.Client("https://api.anthropic.com/mcp")

# List available servers
servers = client.list_servers()

# Connect to a server
async with client.connect(servers[0]) as conn:
    result = await conn.call_tool("web_search", {"query": "MCP protocol"})
    print(result)

Features: Full protocol support, streaming, authentication, retry logic, connection pooling

Version: 1.0+ (stable)

Use case: Production Python applications, data pipelines, automation scripts

Official TypeScript SDK: @modelcontextprotocol/sdk

npm install @modelcontextprotocol/sdk
import { Client } from "@modelcontextprotocol/sdk/client";

const client = new Client({
  name: "my-mcp-client",
  version: "1.0.0",
});

await client.connect({
  transport: "stdio",
  command: "npx",
  args: ["-y", "@my-mcp-server/server"],
});

// Call a tool
const result = await client.callTool({
  name: "calculate",
  arguments: { a: 10, b: 20 },
});

Features: TypeScript-native, full type safety, Node.js + browser support

Version: 0.9+

Use case: Next.js apps, React applications, CLI tools, VS Code extensions

Python MCP Libraries

FastMCP — The Most Popular Community Framework

pip install fastmcp
from fastmcp import FastMCP

mcp = FastMCP("My Server")

@mcp.tool()
def calculate(a: float, b: float, operation: str = "add") -> float:
    """Perform a calculation."""
    if operation == "add":
        return a + b
    elif operation == "multiply":
        return a * b
    return 0

@mcp.resource("config://app-settings")
def get_settings() -> dict:
    """Return application settings."""
    return {"theme": "dark", "debug": True}

Why FastMCP? It dramatically simplifies MCP server creation with a decorator-based API, automatic schema generation, and built-in support for streaming responses.

Stars: 8,400+

Best for: Rapid MCP server development, Python-first teams

TypeScript/JavaScript MCP Libraries

@modelcontextprotocol/sdk (Official)

The official Anthropic SDK covers both client and server implementations:

import { Server } from "@modelcontextprotocol/sdk/server";

const server = new Server(
  { name: "example-server", version: "1.0.0" },
  { capabilities: { tools: {}, resources: {} } }
);

server.setRequestHandler("tools/list", async () => ({
  tools: [{
    name: "greet",
    description: "Greet a user by name",
    inputSchema: {
      type: "object",
      properties: {
        name: { type: "string", description: "The name to greet" }
      }
    }
  }]
}));

server.setRequestHandler("tools/call", async (request) => {
  if (request.params.name === "greet") {
    return { content: [{ type: "text", text: `Hello, ${request.params.arguments.name}!` }] };
  }
});

Go MCP Libraries

go-mcp

go get github.com/metoro-io/mcp-golang
package main

import (
    "context"
    "github.com/metoro-io/mcp-golang"
)

func main() {
    server := mcp.NewServer("my-server", "1.0.0")
    
    server.RegisterTool("calculator", "Perform calculations",
        func(ctx context.Context, args map[string]any) (string, error) {
            a := args["a"].(float64)
            b := args["b"].(float64)
            return fmt.Sprintf("%f", a+b), nil
        })
    
    server.Start()
}

Best for: High-performance Go applications, microservices

Rust MCP Libraries

mcp-rs

cargo add mcp
use mcp::{Server, Tool, Result};

#[tokio::main]
async fn main() -> Result<()> {
    let mut server = Server::new("my-server", "1.0.0");
    
    server.add_tool(Tool::new(
        "hello",
        "Say hello",
        |params: HelloParams| async move {
            Ok(format!("Hello, {}!", params.name))
        },
    )).await?;
    
    server.start().await
}

Best for: Performance-critical applications, embedded systems, WebAssembly

Choosing the Right MCP Library

Quick Decision Guide

  • Building an MCP server in Python? → Use FastMCP for rapid development or try_make_mcp for enterprise-grade templates
  • Building an MCP client in Python? → Use the Official Python SDK
  • Building for the web (TypeScript/JavaScript)? → Use the Official TypeScript SDK + Vercel AI SDK
  • Performance-critical or embedded? → Use mcp-rs
  • Go microservices? → Use go-mcp
  • Enterprise Java? → Use MCP4J

Migration Between Libraries

Most MCP libraries follow the same core concepts — servers, tools, resources, and prompts. Migration is usually straightforward:

# FastMCP → Official Python SDK
# FastMCP decorator style:
@mcp.tool()
def my_tool(arg: str) -> str:
    return f"Result: {arg}"

# Official SDK equivalent:
@server.list_tools()
async def list_tools():
    return [Tool(name="my_tool", description="...", inputSchema={...})]

@server.call_tool()
async def call_tool(name, arguments):
    if name == "my_tool":
        return f"Result: {arguments['arg']}"
