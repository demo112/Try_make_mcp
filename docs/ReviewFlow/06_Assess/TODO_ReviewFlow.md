# TODO_ReviewFlow: 待办事项与优化建议

## 1. 功能增强
- [ ] **动态配置**: 支持从外部 YAML/JSON 文件加载 Prompt 模板，而不是硬编码在 Python 文件中。这样修改 Prompt 不需要重新编译 EXE。
- [ ] **多项目支持**: 目前 `context` 是全局单例，虽然支持 `start_review` 重置，但不支持并发多项目（MCP 协议本身通常是单连接单会话，暂时问题不大）。
- [ ] **更严格的验收**: `submit_work` 目前只检查文件是否存在。可以引入 LLM 自查机制，让 LLM 先读取文件内容自我评估质量，再提交。

## 2. 体验优化
- [ ] **进度可视化**: 增加一个 `get_status` 工具，返回 ASCII 进度条或 Mermaid 状态图。
- [ ] **错误提示**: 当文件检查失败时，提供更具体的错误原因（如“缺少章节X”）。

## 3. 部署优化
- [ ] **自动更新**: EXE 分发机制。
