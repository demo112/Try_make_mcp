# 需求对齐文档：测试用例生成助手

## 1. 原始需求
用户希望构建一个 MCP 服务，核心功能是“写测试用例”。目标用户是局域网内的测试人员。

## 2. 场景分析
测试人员编写用例的痛点通常在于：
- **重复性工作**：对于相似的功能（如 CRUD），用例结构雷同。
- **覆盖率不足**：容易遗漏边界值或异常场景。
- **格式不统一**：不同人员编写的用例风格迥异。

## 3. 解决方案：TestCase Assistant MCP
我们将构建一个专注于“生成”和“管理”测试用例的 MCP 服务。

### 核心功能 (MVP)
1.  **Tool: `generate_test_case_template(feature_name)`**
    - 根据功能名称，生成标准的测试用例模板（包含前置条件、步骤、预期结果）。
2.  **Tool: `recommend_boundary_values(data_type)`**
    - 根据数据类型（如年龄、金额、手机号），推荐测试边界值。
3.  **Resource: `cases://template/standard`**
    - 提供团队统一的用例编写规范/模板文本。

## 4. 关键约束
- **离线运行**：必须能够打包成 `.exe` 独立运行。
- **无外部依赖**：不依赖 OpenAI 等在线 API（因为是离线环境）。
- **逻辑内置**：所有的“智能”生成逻辑必须通过 Python 代码规则实现（Rule-based），或者预置在代码中。

## 5. 交付物
- 源代码 (`src/test_case_server.py`)
- 打包后的可执行文件 (`dist/test-case-assistant.exe`)
- 使用说明文档

# 需求共识文档：TestCase Assistant

## 1. 需求描述
构建一个离线可用的 MCP 服务，辅助测试人员快速编写测试用例。它不依赖大模型（因为是 Server 端），而是提供**规则化**的工具，让 Client 端的大模型（如果局域网内有部署本地大模型）或者直接由用户通过 MCP Client 调用来加速工作。

**特别说明**：MCP Server 本身不包含 LLM。它提供的工具是给 LLM 调用的。如果用户的局域网内没有 LLM，那么这个 MCP Server 将作为“规则库”被用户直接调用（例如通过简单的 CLI 或 GUI 包装）。
*考虑到用户提到“认可大模型”，假设局域网内可能部署了本地大模型（如 Llama3, Qwen 等）或者用户希望通过这个工具展示 MCP 的能力。*

## 2. 验收标准
1.  **独立运行**：提供 `.exe` 文件，在无 Python 环境的 Windows 上运行。
2.  **模板生成**：输入“登录功能”，返回包含正常流和异常流的标准 CSV/Markdown 格式用例。
3.  **边界值推荐**：输入“1-100的整数”，返回 `0, 1, 50, 100, 101, -1, null` 等测试值。
4.  **Mock 数据**：(可选) 生成一些测试用的 Mock 数据。

## 3. 技术实现
- **语言**: Python
- **库**: `mcp`, `faker` (用于生成测试数据)
- **打包**: `pyinstaller`

## 4. 任务边界
- 不包含用例管理系统 (TMS) 的数据库交互。
- 不包含实际执行测试的功能。
