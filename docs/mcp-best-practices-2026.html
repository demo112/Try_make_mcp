1. Error Handling & Resilience

Graceful Degradation

Production MCP servers must handle failures without crashing the AI agent. Implement circuit breakers and fallback responses:

import time
from enum import Enum

class CircuitState(Enum):
    CLOSED = "closed"
    OPEN = "open"
    HALF_OPEN = "half_open"

class CircuitBreaker:
    def __init__(self, failure_threshold=5, timeout=60):
        self.state = CircuitState.CLOSED
        self.failure_count = 0
        self.last_failure_time = None
        self.failure_threshold = failure_threshold
        self.timeout = timeout

    def call(self, func, *args, **kwargs):
        if self.state == CircuitState.OPEN:
            if time.time() - self.last_failure_time > self.timeout:
                self.state = CircuitState.HALF_OPEN
            else:
                return {"error": "Service temporarily unavailable", "fallback": True}
        
        try:
            result = func(*args, **kwargs)
            self._on_success()
            return result
        except Exception as e:
            self._on_failure()
            return {"error": str(e), "fallback": True}

    def _on_success(self):
        self.failure_count = 0
        self.state = CircuitState.CLOSED

    def _on_failure(self):
        self.failure_count += 1
        self.last_failure_time = time.time()
        if self.failure_count >= self.failure_threshold:
            self.state = CircuitState.OPEN
Key insight: Return {"fallback": true} instead of raising exceptions — this lets the AI agent gracefully adapt rather than crash.

2. Rate Limiting & Throttling

Protect your backend services and ensure fair usage with token bucket or sliding window rate limiting:

StrategyProsConsBest For
Token BucketAllows burstsComplex to implementAPI gateways
Sliding WindowFair distributionMemory intensiveUser-level limits
Fixed WindowSimpleBoundary spikesQuick wins
from functools import wraps
import time
import asyncio

class RateLimiter:
    def __init__(self, max_calls: int, period: float):
        self.max_calls = max_calls
        self.period = period
        self.calls = []

    def is_allowed(self) -> bool:
        now = time.time()
        self.calls = [t for t in self.calls if now - t < self.period]
        if len(self.calls) < self.max_calls:
            self.calls.append(now)
            return True
        return False

    def get_retry_after(self) -> float:
        if not self.calls:
            return 0
        return self.period - (time.time() - self.calls[0])

# Usage in MCP tool
limiter = RateLimiter(max_calls=60, period=60)  # 60 req/min

def rate_limited_tool(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        if not limiter.is_allowed():
            return {
                "error": "Rate limit exceeded",
                "retry_after": limiter.get_retry_after()
            }
        return func(*args, **kwargs)
    return wrapper

3. Authentication & Authorization

MCP servers handle sensitive data — implement defense in depth:

Multi-Layer Auth Stack

  1. Transport Layer: HTTPS/TLS mandatory for all production deployments
  2. API Keys: Rotatable keys with scope restrictions
  3. OAuth 2.0: For user-delegated access patterns
  4. mTLS: For high-security internal services
# API Key validation with scope checking
class AuthMiddleware:
    def __init__(self, allowed_scopes: list[str]):
        self.allowed_scopes = allowed_scopes

    def validate_request(self, request, api_key: str) -> dict:
        key_data = self._lookup_key(api_key)
        if not key_data:
            return {"authorized": False, "error": "Invalid API key"}
        
        if not key_data.get("active"):
            return {"authorized": False, "error": "API key revoked"}
        
        required_scope = request.get("required_scope")
        if required_scope and required_scope not in key_data["scopes"]:
            return {"authorized": False, "error": "Insufficient permissions"}
        
        return {"authorized": True, "user_id": key_data["user_id"]}
Security tip: Never log API keys or tokens. Hash them before storage and use constant-time comparison for validation.

4. Structured Logging

Production debugging requires structured, queryable logs:

import json
import logging
from datetime import datetime

class StructuredLogger:
    def __init__(self, service_name: str):
        self.service = service_name
        self.logger = logging.getLogger(service_name)
        self.logger.setLevel(logging.INFO)

    def log(self, level: str, event: str, **kwargs):
        log_entry = {
            "timestamp": datetime.utcnow().isoformat() + "Z",
            "service": self.service,
            "level": level.upper(),
            "event": event,
            **kwargs
        }
        getattr(self.logger, level.lower())(json.dumps(log_entry))

# Usage
log = StructuredLogger("mcp-server")
log.log("info", "tool_called", 
    tool="file_reader", 
    user="user_123", 
    duration_ms=45,
    success=True
)
# Output: {"timestamp":"2026-05-13T14:30:00Z","service":"mcp-server","level":"INFO","event":"tool_called","tool":"file_reader","user":"user_123","duration_ms":45,"success":true}

# tests/test_mcp_tools.py
import pytest
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("test-server")

@mcp.tool()
def add_numbers(a: int, b: int) -> int:
    return a + b

class TestMCPTools:
    def test_add_numbers_basic(self):
        result = add_numbers(2, 3)
        assert result == 5

    def test_add_numbers_negative(self):
        result = add_numbers(-1, 1)
        assert result == 0

    @pytest.mark.asyncio
    async def test_mcp_protocol_integration(self):
        # Test the full MCP protocol round-trip
        async with mcp.client_session() as session:
            await session.initialize()
            result = await session.call_tool("add_numbers", {"a": 10, "b": 20})
            assert result.content[0].text == "30"

7. Production Checklist

Before going to production, verify each of these:

  • ✅ Health endpoint: GET /health returns 200 with status JSON
  • ✅ Graceful shutdown: Handle SIGTERM, drain connections
  • ✅ Structured logging: JSON format, no PII logged
  • ✅ Rate limiting: Token bucket or sliding window
  • ✅ Auth validation: Every request authenticated
  • ✅ Circuit breaker: External API calls protected
  • ✅ Metrics exposed: Prometheus / OpenTelemetry compatible
  • ✅ Dockerfile ready: Multi-stage build, non-root user
  • ✅ Environment config: No secrets in code
  • ✅ Test coverage: 80%+ for critical paths
  • ✅ Documentation: OpenAPI spec or MCP schema
  • ✅ Monitoring: Error rate, latency, availability alerts
