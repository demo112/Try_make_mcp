Prerequisites

Before you start, make sure you have:

  • Python 3.9+ — python3 --version
  • Git — git --version
  • Claude Desktop or any MCP-compatible client (optional, for live testing)
  • Docker (optional, for containerized deployment)
💡 No prior MCP experience needed. Try_make_mcp is designed for developers who want to build MCP servers without understanding the full protocol internals. The factory handles the boilerplate.

Install the Factory

Clone the Try_make_mcp repository:

git clone https://github.com/demo112/Try_make_mcp.git
cd Try_make_mcp
pip install -e .

Verify installation:

python -c "from mcp_factory import *; print('Factory ready ✓')"

Choose a Template

Pick the template closest to your use case. For this tutorial, we'll build a Web Scraper MCP.

cd Try_make_mcp
python -m mcp_factory.scaffold --template web_scraper --name my-scraper

This creates apps/my_scraper/ with:

  • server.py — MCP server implementation
  • config.py — Configuration and resources
  • requirements.txt — Dependencies
  • test_server.py — Unit tests
  • Dockerfile — Container build

Customize Resources & Tools

Open apps/my_scraper/config.py and define your MCP resources:

# Define MCP resources (files, URLs, API endpoints)
resources = [
    {
        "name": "page_content",
        "uri": "https://example.com",
        "mimeType": "text/html",
        "description": "Web page HTML content"
    }
]

# Define MCP tools
tools = [
    {
        "name": "scrape_url",
        "description": "Scrape content from a URL",
        "inputSchema": {
            "type": "object",
            "properties": {
                "url": {"type": "string", "description": "Target URL"}
            },
            "required": ["url"]
        }
    }
]

Implement the tool in server.py:

@mcp_tool(name="scrape_url", description="Scrape content from a URL")
def scrape_url(url: str) -> dict:
    """Scrape and return content from target URL."""
    import urllib.request
    from html.parser import HTMLParser

    class TextExtractor(HTMLParser):
        def __init__(self):
            super().__init__()
            self.text = []
            self.in_body = False

        def handle_starttag(self, tag, attrs):
            if tag == 'body':
                self.in_body = True

        def handle_data(self, data):
            if self.in_body:
                self.text.append(data.strip())

    req = urllib.request.Request(url, headers={'User-Agent': 'Mozilla/5.0'})
    with urllib.request.urlopen(req) as response:
        html = response.read().decode('utf-8', errors='ignore')

    parser = TextExtractor()
    parser.feed(html)
    return {"url": url, "content": '\n'.join(parser.text[:500])}

Run Local Tests

Run the built-in test suite:

cd apps/my_scraper
python -m pytest test_server.py -v
✅ CI/CD is automatic. Every push triggers GitHub Actions with lint checks, unit tests, and integration validation. Your MCP server is tested on every commit.

Deploy with Docker

Build and run your MCP server in a container:

cd apps/my_scraper
docker build -t my-scraper-mcp .
docker run -p 8000:8000 my-scraper-mcp

Or use the MCP Transport to connect directly to Claude Desktop. Add this to your claude_desktop_config.json:

{
  "mcpServers": {
    "my-scraper": {
      "command": "docker",
      "args": ["run", "--rm", "-i", "my-scraper-mcp"]
    }
  }
}

Publish & Share

When your MCP server is ready, share it with the community:

  1. Submit to Glama — Get a quality score and join the MCP registry at glama.ai
  2. Add to awesome-mcp-servers — Submit a PR to the community list
  3. Publish on PyPI — python -m build && twine upload dist/*
  4. Add a LICENSE — MIT is recommended for open collaboration
