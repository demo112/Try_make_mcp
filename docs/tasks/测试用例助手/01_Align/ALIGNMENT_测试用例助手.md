# 需求对齐文档：测试用例生成助手

## 1. 原始需求
用户希望构建一个 MCP 服务，核心功能是“写测试用例”。目标用户是局域网内的测试人员。

## 2. 场景分析
测试人员编写用例的痛点通常在于：
- **重复性工作**：对于相似的功能（如 CRUD），用例结构雷同。
- **覆盖率不足**：容易遗漏边界值或异常场景。
- **格式不统一**：不同人员编写的用例风格迥异。

## 3. 解决方案：TestCase Assistant MCP
我们将构建一个专注于“生成”和“管理”测试用例的 MCP 服务。

### 核心功能 (MVP)
1.  **Tool: `generate_test_case_template(feature_name)`**
    - 根据功能名称，生成标准的测试用例模板（包含前置条件、步骤、预期结果）。
2.  **Tool: `recommend_boundary_values(data_type)`**
    - 根据数据类型（如年龄、金额、手机号），推荐测试边界值。
3.  **Resource: `cases://template/standard`**
    - 提供团队统一的用例编写规范/模板文本。

## 4. 关键约束
- **离线运行**：必须能够打包成 `.exe` 独立运行。
- **无外部依赖**：不依赖 OpenAI 等在线 API（因为是离线环境）。
- **逻辑内置**：所有的“智能”生成逻辑必须通过 Python 代码规则实现（Rule-based），或者预置在代码中。

## 5. 交付物
- 源代码 (`src/test_case_server.py`)
- 打包后的可执行文件 (`dist/test-case-assistant.exe`)
- 使用说明文档
