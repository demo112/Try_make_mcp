MCP with Claude Desktop: Complete Setup Guide (2026)

Reading time: 12 min Updated: 2026-05 Level: Beginner to Advanced

The Model Context Protocol (MCP) is revolutionizing how AI assistants like Claude interact with external tools and data sources. This guide walks you through setting up MCP servers with Claude Desktop on Windows, macOS, and Linux — from zero to a fully operational setup in under 30 minutes.

What is Claude Desktop MCP Integration?

Claude Desktop is Anthropic's desktop application that brings Claude AI to your local environment. By integrating MCP servers, you can extend Claude's capabilities to:

  • Read and write local files with custom permissions
  • Query databases (PostgreSQL, SQLite, MySQL)
  • Search codebases and execute terminal commands
  • Connect to cloud APIs (GitHub, Slack, Notion, Figma)
  • Run custom AI-powered workflows built with Try_make_mcp
💡 Pro Tip
MCP acts as a standardized bridge between Claude and your tools. Instead of writing custom integrations for every tool, MCP provides a universal protocol — similar to how USB-C works for hardware connections.

Prerequisites

Before starting, ensure you have:

  • Claude Desktop installed (download from claude.ai/desktop)
  • Node.js v18+ (for running TypeScript MCP servers)
  • Python 3.9+ (for Python-based MCP servers)
  • Git installed and configured

Step 1: Install Claude Desktop

Download and install Claude Desktop from the official website:

# macOS — Download from https://claude.ai/desktop
# Or use Homebrew:
brew install --cask claude

# Windows — Download installer from claude.ai/desktop
# Linux (Debian/Ubuntu):
curl -fsSL https://downloads.anthropic.com/desktop/linux-install.sh | sh
Note
Claude Desktop is separate from the web interface. The MCP integration only works with the desktop application, as it requires local process communication.

Step 2: Locate the Claude Desktop Configuration File

Claude Desktop uses a JSON configuration file to manage MCP server connections. You need to edit this file to add new servers.

# macOS/Linux
~/Library/Application Support/Claude/claude_desktop_config.json

# Windows
%APPDATA%\Claude\claude_desktop_config.json
⚠️ Backup First
Always make a backup of your config file before editing: cp claude_desktop_config.json claude_desktop_config.json.bak

Step 3: Configure Your First MCP Server

Open the configuration file in your preferred editor. The structure looks like this:

{
  "mcpServers": {
    "server-name": {
      "command": "npx",
      "args": ["-y", "@some/mcp-server-package"]
    }
  }
}

Example 1: Filesystem MCP Server

Connect Claude to your local filesystem with controlled access:

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem"],
      "env": {
        "allowedDirectories": ["~/Documents", "~/Desktop"]
      }
    }
  }
}

Example 2: GitHub MCP Server

Give Claude access to your GitHub repositories:

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "your-token-here"
      }
    }
  }
}
🔐 Security
MCP servers run with the permissions granted in the configuration. Only enable servers for tools you trust. Review the MCP Authentication & Security Guide before connecting sensitive services.

Step 4: Using MCP Servers in Claude Desktop

After saving your configuration, restart Claude Desktop. The MCP servers will appear in the tool selection panel.

Advanced Configuration

Running MCP Servers via Docker

For isolated, reproducible environments, run MCP servers inside Docker containers:

{
  "mcpServers": {
    "document-converter": {
      "command": "docker",
      "args": [
        "run", "--rm", "-i",
        "--mount", "type=bind,src=/path/to/docs,dst=/app/docs,ro",
        "ghcr.io/demo112/md-converter:latest"
      ]
    }
  }
}

Environment Variables for MCP Servers

{
  "mcpServers": {
    "custom-api": {
      "command": "python",
      "args": ["/path/to/mcp_server.py"],
      "env": {
        "API_KEY": "your-api-key",
        "API_URL": "https://api.example.com",
        "LOG_LEVEL": "INFO"
      }
    }
  }
}

Multi-Server Configuration

You can run multiple MCP servers simultaneously:

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem"],
      "env": { "allowedDirectories": ["~/Projects"] }
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxx" }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "DATABASE_URL": "postgresql://user:pass@localhost:5432/mydb"
      }
    }
  }
}

Troubleshooting Common Issues

Issue: MCP server not starting

Check the Claude Desktop logs for errors:

# macOS
~/Library/Logs/Claude/claude_desktop.log

# Windows
%APPDATA%\Claude\logs\
Debug Tip
Run the MCP server command directly in your terminal first to verify it works outside of Claude Desktop. This helps identify if the issue is with the server itself or the Claude integration.

Issue: "Command not found" errors

  • Ensure npx is in your PATH
  • For Python servers, use the full path: "command": "/usr/local/bin/python3"
  • For Docker, ensure Docker Desktop is running

Issue: Permission denied on macOS

# Allow Node.js to run unsigned scripts
sudo xattr -rd com.apple.quarantine /path/to/npx

Best Practices for Claude Desktop MCP Setup

  1. Start with minimal permissions — Only grant access to directories and resources you actually need
  2. Use environment variables — Never hardcode API keys in config; use env vars or a secrets manager
  3. Test outside Claude first — Verify MCP servers work independently before connecting to Claude
  4. Keep servers updated — Run npx -y @package-name@latest periodically
  5. Monitor logs — Regularly check Claude Desktop logs for any unexpected behavior
  6. Use Docker for isolation — Containerize MCP servers to avoid dependency conflicts
