# L1 层策略审视报告 (L1 Strategy Review)

## 1. 策略定义
L1 (业务流程层) 当前的核心策略是 **"文档即接口 (Document as Interface)"**。
用户通过编写 Markdown 文档提出问题，MCP 服务通过 RAG 技术在文档中"填空"或"批注"答案。

## 2. 优势分析 (Pros)
1. **零切换成本**: 用户无需离开 IDE 或文档编辑器，直接在工作流中获得 AI 辅助。
2. **上下文完整**: 相比于 Chat 窗口的碎片化问答，基于文档的问答天然具备完整的项目上下文（如前文的方案描述）。
3. **非侵入式**: 采用"影子文件 (Shadow Copy)"机制，确保 AI 的建议不会破坏用户已有的心血，降低了用户试错的心理门槛。

## 3. 风险与不足 (Cons)
1. **"盲信"风险**:
   - 当前的呈现方式（直接插入文本）可能让用户误以为这是"定论"而非"建议"。
   - 如果置信度提示不够醒目，用户可能忽略低质量的回答。
2. **反馈闭环缺失**:
   - 当用户修改了 AI 的建议后，这些修改目前仅停留在文档上。
   - 系统缺乏机制去"感知"用户的修正，导致同样的错误可能重复出现（除非启用 Scenario 3 的知识收割）。
3. **提问粒度依赖**:
   - 效果高度依赖用户的问题描述。如果问题过于宽泛（如"安全性如何？"），RAG 可能检索出大量无关信息。

## 4. 改进建议 (Recommendations)
1. **可视化增强**:
   - 在生成的 Markdown 中使用更直观的标识（如 🔴/🟡/🟢 图标）来代表置信度，而不仅仅是数字。
2. **交互式反馈**:
   - 引入简单的"反馈标记"（如在建议后附带 `[Accept] / [Reject]` 链接或命令），让用户能显式反馈结果，为未来的强化学习积累数据。
3. **主动引导**:
   - 当检测到问题模糊时，AI 不应强行回答，而应在文档中插入"追问"（Clarification Request），引导用户细化问题。
