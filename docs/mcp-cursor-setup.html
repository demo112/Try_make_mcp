MCP with Cursor: Complete Setup Guide (2026)

Published: May 2026 | Reading time: 14 minutes | Category: Setup

What is Cursor IDE and Why Use MCP with It?

Cursor is an AI-first code editor built on VS Code, designed for developers who want intelligent code completion, chat, and refactoring powered by large language models. It has rapidly gained adoption among individual developers and small teams since its public launch.

The Model Context Protocol (MCP) extends Cursor's capabilities by allowing it to connect to external tools and data sources. While Cursor has its own built-in AI features, MCP integration unlocks additional specialized tools — from RAG document search to custom API integrations — directly within the editor workflow.

Prerequisites

  • Cursor version 0.45+ (download from cursor.com)
  • Node.js 18+ (for running npm-based MCP servers)
  • Python 3.9+ (for Python-based MCP servers)
  • Git installed and configured
  • A terminal/command prompt

Method 1: Using Cursor's Built-in MCP Support

Cursor 0.45+ includes native MCP support. Here's how to configure it:

Step 1: Locate Cursor's MCP Settings

  1. Open Cursor → Cmd/Ctrl + Shift + P
  2. Type MCP and select "MCP: Configure"
  3. This opens .cursor/mcp.json in your home/user directory

Step 2: Configure a Filesystem MCP Server

Add this configuration to enable local file access:

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/your/project"]
    }
  }
}

Step 3: Configure GitHub MCP Server

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "your_token_here"
      }
    }
  }
}

Troubleshooting Common Issues

Issue 1: MCP Server Won't Start

Symptom: "Connection refused" or timeout errors

Solution:

  • Verify Node.js/Python are installed: node --version / python --version
  • Check the server command is correct (try running it manually in terminal)
  • Ensure the working directory (cwd) is set correctly

Issue 2: "MCP: Command not found" Error

Symptom: Cursor shows "command not found" for npx commands

Solution: Install the MCP server globally or use the full path to npx:

# Install globally
npm install -g @modelcontextprotocol/server-filesystem

# Then in mcp.json, use:
"command": "mcp-server-filesystem"

Issue 3: Authentication Failures

Symptom: GitHub MCP returns 401 errors

Solution: Generate a new Personal Access Token at github.com/settings/tokens with appropriate scopes (repo, read:user).

Issue 4: Cursor Doesn't Detect MCP Servers

Symptom: MCP servers don't appear in Cursor's command palette

Solution:

  1. Restart Cursor completely (Cmd+Q, then reopen)
  2. Check ~/.cursor/mcp.json for syntax errors (valid JSON)
  3. Ensure no duplicate server names

Best Practices for Cursor + MCP

  • Security first: Never expose sensitive API keys in the mcp.json file that gets committed to version control. Use environment variables.
  • Start simple: Begin with one MCP server, verify it works, then add more.
  • Test outside Cursor: Run the MCP server manually first to catch configuration errors.
  • Use Try_make_mcp: The factory pattern makes it easy to create project-specific MCP servers without starting from scratch.
  • Monitor token usage: MCP tool calls consume context tokens. Be mindful in long conversations.
  • Document your setup: Keep a README in your project documenting which MCP servers are configured.
